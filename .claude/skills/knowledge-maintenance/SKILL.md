# Knowledge Maintenance

触发：用户要求 / STATUS.md 中 total_files>=20 且距 last_full_maintenance>30 天。

## 流程

### 1. 扫描

读 `./` 所有知识文件（排除元文件和 journal），解析 frontmatter 构建清单。

### 2. 检测

- **结构：** 缺必填字段 / category 与目录不匹配 / related 缺 type
- **内容：** 疑似重复 / 过时(>180天+时效标签) / 孤岛(无关联) / 单薄(<3句) / 已更新无 revision
- **关联：** 死链 / 单向 / GRAPH.md 不一致
- **标签：** 未注册 / 疑似同义 / 条目数不准

### 3. 自动修复

移除死链 → 补全单向关联(type 设"关联") → 重建 INDEX.md(分类+标签+高优) → 重建 GRAPH.md(按领域拆子图，边标签用 type) → 同步 TAGS.md 条目数 → 未注册标签自动注册(标"待审核")

### 4. Journal 归档

>90 天日志按季度合并到 archive/YYYY-QN.md，仅留"输入+写入"，删原文件。

### 5. 需确认

- **标签合并：** 展示同义对 → 确认 → 更新文件 tags + TAGS.md
- **条目合并：** 展示重复对 → 确认 → 合并正文/tags/related，更新引用，删被合并文件
- **单薄文件：** 建议合并到父主题或充实

### 6. 报告

```
统计：总条目 | 分类分布 | 高优 | 标签 Top5 | 关联数(均值) | 孤岛率
修复：死链 | 补关联 | 注册标签 | 归档日志 | 重建 INDEX/GRAPH
待处理：重复 | 同义标签 | 单薄 | 孤岛 | 过时
覆盖：领域分布 | 近30天活跃标签 | 薄弱区域
```

### 7. 更新 STATUS.md

重置计数器 → 更新维护时间线 → 未处理问题写入待处理队列 → 清除已解决项。

### 8. 写 journal

记录本次维护摘要。
